## Overview

Filter-def supports 10 primitive filter types that work across all adapters. Each filter type has a specific `kind` and expected input type.

## Equality Filters

### eq (Equals)

Passes when the field value equals the filter input value.

```typescript
interface EqFilter<Entity> {
  kind: 'eq';
  field?: FieldPath<Entity>;
}
```

<CodeGroup>
```typescript In-Memory
const userFilter = inMemoryFilter<User>().def({
  name: { kind: 'eq' },
  role: { kind: 'eq' },
});

const admins = users.filter(userFilter({ role: 'admin' }));
```

```typescript Drizzle
const userFilter = drizzleFilter(usersTable).def({
  name: { kind: 'eq' },
  role: { kind: 'eq' },
});

const where = userFilter({ role: 'admin' });
await db.select().from(usersTable).where(where);
```

```typescript BigQuery
const userFilter = bigqueryFilter<User>().def({
  name: { kind: 'eq' },
  role: { kind: 'eq' },
});

const { sql, params } = userFilter({ role: 'admin' });
// SQL: role = @role
```
</CodeGroup>

### neq (Not Equals)

Passes when the field value does not equal the filter input value.

```typescript
interface NeqFilter<Entity> {
  kind: 'neq';
  field?: FieldPath<Entity>;
}
```

<CodeGroup>
```typescript In-Memory
const userFilter = inMemoryFilter<User>().def({
  role: { kind: 'neq' },
});

const nonAdmins = users.filter(userFilter({ role: 'admin' }));
```

```typescript Drizzle
const userFilter = drizzleFilter(usersTable).def({
  role: { kind: 'neq' },
});

const where = userFilter({ role: 'admin' });
await db.select().from(usersTable).where(where);
```
</CodeGroup>

## String Filters

### contains (String Contains)

Passes when the string field contains the filter input substring.

```typescript
interface ContainsFilter<Entity> {
  kind: 'contains';
  field?: FieldPath<Entity>;
  caseInsensitive?: boolean;
}
```

<Note>
The `caseInsensitive` option enables case-insensitive matching. In Drizzle, this uses `ILIKE` instead of `LIKE`.
</Note>

<CodeGroup>
```typescript In-Memory
const productFilter = inMemoryFilter<Product>().def({
  nameContains: { kind: 'contains', field: 'name' },
  emailDomain: { kind: 'contains', field: 'email', caseInsensitive: true },
});

const laptops = products.filter(
  productFilter({ nameContains: 'Laptop' })
);
```

```typescript Drizzle
const userFilter = drizzleFilter(usersTable).def({
  emailDomain: { kind: 'contains', field: 'email', caseInsensitive: true },
});

const where = userFilter({ emailDomain: '@company.com' });
// Generates: email ILIKE '%@company.com%'
```

```typescript BigQuery
const userFilter = bigqueryFilter<User>().def({
  nameContains: { kind: 'contains', field: 'name' },
  emailDomain: { kind: 'contains', field: 'email', caseInsensitive: true },
});

const { sql, params } = userFilter({ emailDomain: '@company.com' });
// SQL: LOWER(email) LIKE LOWER(@emailDomain)
// params: { emailDomain: '%@company.com%' }
```
</CodeGroup>

## Array Filters

### inArray (In Array)

Passes when the field value is contained in the filter input array.

```typescript
interface InArrayFilter<Entity> {
  kind: 'inArray';
  field?: FieldPath<Entity>;
}
```

**Input Type**: Array of field values

<CodeGroup>
```typescript In-Memory
const userFilter = inMemoryFilter<User>().def({
  role: { kind: 'inArray' },
  status: { kind: 'inArray' },
});

const staffUsers = users.filter(
  userFilter({ role: ['admin', 'moderator', 'editor'] })
);
```

```typescript Drizzle
const userFilter = drizzleFilter(usersTable).def({
  role: { kind: 'inArray' },
});

const where = userFilter({ role: ['admin', 'moderator'] });
await db.select().from(usersTable).where(where);
```

```typescript BigQuery
const userFilter = bigqueryFilter<User>().def({
  role: { kind: 'inArray' },
});

const { sql, params } = userFilter({ role: ['admin', 'moderator'] });
// SQL: role IN UNNEST(@role)
// params: { role: ['admin', 'moderator'] }
```
</CodeGroup>

## Null Filters

### isNull (Is Null)

Passes when the field value is `null` or `undefined`.

```typescript
interface IsNullFilter<Entity> {
  kind: 'isNull';
  field?: FieldPath<Entity>;
}
```

**Input Type**: `boolean`

<CodeGroup>
```typescript In-Memory
const userFilter = inMemoryFilter<User>().def({
  hasPhone: { kind: 'isNull', field: 'phone' },
});

// Find users without phone numbers
const noPhone = users.filter(userFilter({ hasPhone: true }));

// Find users with phone numbers (inverted)
const withPhone = users.filter(userFilter({ hasPhone: false }));
```

```typescript Drizzle
const userFilter = drizzleFilter(usersTable).def({
  hasPhone: { kind: 'isNull', field: 'phone' },
});

const where = userFilter({ hasPhone: true });
// Generates: phone IS NULL
```
</CodeGroup>

### isNotNull (Is Not Null)

Passes when the field value is not `null` and not `undefined`.

```typescript
interface IsNotNullFilter<Entity> {
  kind: 'isNotNull';
  field?: FieldPath<Entity>;
}
```

**Input Type**: `boolean`

<CodeGroup>
```typescript In-Memory
const userFilter = inMemoryFilter<User>().def({
  hasPhone: { kind: 'isNotNull', field: 'phone' },
});

// Find users with phone numbers
const withPhone = users.filter(userFilter({ hasPhone: true }));

// Find users without phone numbers (inverted)
const noPhone = users.filter(userFilter({ hasPhone: false }));
```

```typescript Drizzle
const userFilter = drizzleFilter(usersTable).def({
  hasPhone: { kind: 'isNotNull', field: 'phone' },
});

const where = userFilter({ hasPhone: true });
// Generates: phone IS NOT NULL
```
</CodeGroup>

<Info>
Both `isNull` and `isNotNull` accept a boolean input. When `true`, the filter applies as expected. When `false`, the logic is inverted.
</Info>

## Comparison Filters

Comparison filters work with any comparable data type: numbers, strings, dates, etc.

```typescript

  kind: 'gt';
  field?: FieldPath<Entity>;
```

<CodeGroup>

```typescript Drizzle

```
</CodeGroup>

## Range Filters

Combine comparison filters to create range filters:

```typescript
const productFilter = inMemoryFilter<Product>().def({
  minPrice: { kind: 'gte', field: 'price' },
  maxPrice: { kind: 'lte', field: 'price' },
});

// Find products between $100 and $500
const midRange = products.filter(
  productFilter({ minPrice: 100, maxPrice: 500 })
);
```

<CodeGroup>
```typescript Date Ranges
const eventFilter = inMemoryFilter<Event>().def({
  startDate: { kind: 'gte', field: 'date' },
  endDate: { kind: 'lte', field: 'date' },
});

const events = allEvents.filter(eventFilter({
  startDate: new Date('2024-01-01'),
  endDate: new Date('2024-12-31'),
}));
```

```typescript Age Ranges
const userFilter = inMemoryFilter<User>().def({
  minAge: { kind: 'gte', field: 'age' },
  maxAge: { kind: 'lte', field: 'age' },
});

const targetDemo = users.filter(userFilter({ minAge: 25, maxAge: 45 }));
```
</CodeGroup>

## Filter Type Reference

| Kind | Description | Input Type | Example |
|------|-------------|------------|----------|
| `eq` | Exact equality | Field type | `{ name: 'John' }` |
| `neq` | Not equal | Field type | `{ role: 'guest' }` |
| `contains` | String contains substring | `string` | `{ emailContains: '@company.com' }` |
| `inArray` | Value in array | `Field type[]` | `{ role: ['admin', 'moderator'] }` |
| `isNull` | Is null/undefined | `boolean` | `{ hasPhone: true }` |
| `isNotNull` | Is not null/undefined | `boolean` | `{ hasEmail: true }` |
| `gt` | Greater than | Field type | `{ minPrice: 100 }` |
| `gte` | Greater than or equal | Field type | `{ minAge: 18 }` |
| `lt` | Less than | Field type | `{ maxPrice: 500 }` |
| `lte` | Less than or equal | Field type | `{ maxAge: 65 }` |
